Personvernerklæring

Sist oppdatert: 1. april 2026

Bumle Bjørn er utviklet av Jonas Haug. Denne personvernerklæringen forklarer hvordan appen håndterer data. Vi tar personvern på alvor — spesielt fordi appen er laget for barn.

1. Datainnsamling

Bumle Bjørn samler ikke inn personopplysninger. Vi har ingen brukerkontoer, ingen registrering og ingen pålogging.

All data som appen bruker lagres lokalt på enheten og eventuelt i brukerens egen iCloud-konto:

  • Barneprofiler: Navn, alder, avatar og læringsframgang lagres i UserDefaults og synkroniseres via iCloud Key-Value Store til brukerens egne enheter.
  • Lydopptak: Stemmeopptak som foreldre spiller inn lagres kun lokalt på enheten. De synkroniseres ikke med iCloud eller noen ekstern tjeneste.
  • Egne bilder: Profilbilder og egne kategoribilder lagres lokalt på enheten.
  • Innstillinger: Brukervalg som språk, vanskelighetsgrad og skjermtid lagres lokalt.

2. Tredjeparter

Bumle Bjørn bruker ingen tredjeparts SDK-er, analysetjenester, annonsenettverk eller sporingsverktøy. Vi deler ingen data med tredjeparter.

3. Annonsering og sporing

Appen inneholder ingen reklame og ingen sporing. Vi bruker ikke IDFA, fingerprinting eller andre identifiseringsmetoder.

4. Abonnementer

Bumle+ er et valgfritt abonnement som håndteres av Apple gjennom App Store. Vi har ikke tilgang til betalingsinformasjon. Abonnementsstatus sjekkes via StoreKit og lagres lokalt.

5. Barn og COPPA

Bumle Bjørn er designet for barn i alderen 1–8 år. Appen:

  • Samler ikke inn personlig identifiserbar informasjon fra barn
  • Inneholder ikke lenker til sosiale medier eller eksterne nettsider
  • Har foreldrekontroll med PIN-kode og Face ID
  • Tillater ikke barn å dele informasjon
  • Inneholder ikke reklame

6. Datakontroll

Foreldre har full kontroll over all data:

  • Sletting: All data kan slettes permanent via Innstillinger → Om Bumle Bjørn → Slett alle data.
  • Lydopptak: Individuelle opptak kan slettes via lydinnstillingene.
  • Profiler: Profiler kan slettes enkeltvis i sidemenyen.

7. Sikkerhet

  • PIN-koder lagres som SHA256-hash i iOS Keychain
  • Lydopptak er kryptert med CryptoKit
  • Filer er beskyttet med iOS File Protection
  • Foreldrelås krever PIN og/eller Face ID

8. Endringer

Vi kan oppdatere denne personvernerklæringen ved behov. Endringer vil bli publisert på denne siden med oppdatert dato.
